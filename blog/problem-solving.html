Technical Blog - Problem Solving

When blocked on a simple problem...

While working on the fizzbuzz exercise, I was unable to pass the tests but did not know why as I thought my code was correct.

In my problem solving approach, I tried the rubber ducky debuggin method, where I talked through my solution, and the problem outloud as if talking to someone. Finally, I used google, which was when I realised I was not 'returning' the solution, but instead assigning the solution to a variable.

Throughout the process, I felt a little inadequate as I could not see the issue on my own. After finding the answer, I felt a little silly.

However, I think this was due to my lack of experience coding and I hope to keep practicing writing code in order to get better.

I am learning to use the resources available. I used to be quite against using chatGPT, for example. However, I find it very helpful to debug when really stuck. It can also give you tips, wihtout spoiling the answer if you ask for that.

I am also now very aware that you need to 'return' your answer, and am in the process of knowing when to do what.

Lastly, I think I am still on somewhat of a fixed mindset so I feel like now that I know this, I can put more effort to changing my mindset from "I can't do this task on my own" to "I can't do this task on my own yet".

Reflect on a time you were reluctant to ask for help. Consider what made you reluctant to do so. What might you try differently next time?

During the 'manipulating objects' section, I was really struggling as I was getting an error relating to 'Jest' on my terminal when trying to run my code to test it. I tried to fix the error myself using Google and ChatGPT. I was hesitant to ask on Discord as it was almost 8pm and it is always a bit scary for me to post on there. However, I felt like I had to other choice, so I did ask on discord.

I was frustrated, as due to this error, I thought I couldn't even move on to the next tasks.

After waiting for a reply for a while, I decided to calm down, stop panicking, and think about how I could fix it on my own. As the terminal told me where the issues where on my code - the lines we are not supposed to touch! - I went on gitHub to find the original code from the Dev Academy. There, I found that the lines above and below my code had changed from when I first forked them. I promptly copied and pasted and my issue was solved!

I think I would be less reluctant to ask for help if I was not in a state of panic and was able to exhaust all other options before doing so. I enjoy problem solving instead of being given answers, so I think this may also be attributed to my 'fixed mindset'. I do not have a 'growth mindset' YET.
